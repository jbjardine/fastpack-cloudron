# Custom App Example: Deploy a Python Web App

This guide walks through deploying a **custom Python web application** to Cloudron. Not a pre-built Docker image — your own code.

## What We Are Building

A simple Python Flask app with:
- A web interface (HTTP port 8000)
- Persistent storage (localstorage addon)
- Cloudron SSO (OIDC addon)

**Objective**: Show the full flow from code to deployed Cloudron app.

## Project Structure

```
my-flask-app/
├── app.py              # Your Flask application
├── requirements.txt    # Python dependencies
└── templates/
    └── index.html      # HTML template
```

### app.py

```python
from flask import Flask, render_template
import os

app = Flask(__name__)

@app.route('/')
def home():
    name = os.environ.get('CLOUDRON_OIDC_DISPLAY_NAME', 'Guest')
    return render_template('index.html', name=name)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000)
```

### requirements.txt

```
flask==3.1.0
gunicorn==23.0.0
```

### templates/index.html

```html
<!DOCTYPE html>
<html>
<body>
  <h1>Hello, {{ name }}!</h1>
  <p>This app is running on Cloudron.</p>
</body>
</html>
```

## Step 1: Generate the Cloudron Package

1. Go to [FastPackCloudron](https://jbjardine.github.io/fastpack-cloudron/)
2. Fill in the form:

| Field | Value |
|-------|-------|
| Docker Image | `python:3.12-slim` |
| App Title | `My Flask App` |
| Stack | Python |
| Web Interface | Yes |
| HTTP Port | `8000` |
| SSO | OIDC |
| Storage | localstorage (checked) |

3. Click **Download ZIP**

## Step 2: Customize the Generated Files

Extract the ZIP. You will see:

```
my-flask-app-cloudron/
├── CloudronManifest.json
├── Dockerfile
├── start.sh
├── .dockerignore
├── ...
```

### Edit the Dockerfile

Open `Dockerfile` and add your app code. The generated Dockerfile already has the base structure. Add your code after the `COPY` section:

```dockerfile
FROM python:3.12-slim

# Cloudron conventions: non-root user
RUN apt-get update && apt-get install -y --no-install-recommends gosu \
    && rm -rf /var/lib/apt/lists/* \
    && groupadd -r cloudron -g 808 \
    && useradd -r -g cloudron -u 808 -d /app cloudron

# Install Python dependencies
COPY requirements.txt /app/code/requirements.txt
RUN pip install --no-cache-dir -r /app/code/requirements.txt

# Copy your application code
COPY app.py /app/code/app.py
COPY templates/ /app/code/templates/

# Copy Cloudron files
COPY start.sh /app/code/start.sh
RUN chmod +x /app/code/start.sh

EXPOSE 8000

CMD ["/app/code/start.sh"]
```

### Edit start.sh

The generated `start.sh` already handles localstorage init. Replace the app command:

```bash
#!/bin/bash
set -eu

# Initialize localstorage on first install
if [[ ! -f /app/data/.initialized ]]; then
    echo "=> First run: initializing storage"
    touch /app/data/.initialized
fi
chown -R cloudron:cloudron /app/data

# Start the app as the cloudron user
cd /app/code
exec gosu cloudron:cloudron gunicorn \
    --bind 0.0.0.0:8000 \
    --workers 2 \
    --access-logfile - \
    app:app
```

### Copy Your Code Into the Package

Copy your application files into the same folder:

```
my-flask-app-cloudron/
├── CloudronManifest.json   # Generated by FastPackCloudron
├── Dockerfile              # Edited above
├── start.sh                # Edited above
├── .dockerignore
├── app.py                  # YOUR CODE
├── requirements.txt        # YOUR CODE
├── templates/              # YOUR CODE
│   └── index.html
└── fastpack-deploy-*       # Deploy binary (from Step 3)
```

> **Important**: The deploy binary only uploads files from a strict allow-list (Dockerfile, start.sh, etc.), not your app code. Your code is included because it's referenced in the Dockerfile's `COPY` instruction — Docker pulls it during the build.

Wait — actually, the Build Service needs ALL your source files in the tarball. The current Go CLI has a strict allow-list and will NOT include `app.py` or `templates/`.

**Workaround for custom apps**: Use the `cloudron build` CLI instead:

```bash
# Option A: Use the Cloudron CLI (full flexibility)
npm install -g cloudron
cloudron build --set-build-service devtools.your-domain.com
cloudron build --repository devtools.your-domain.com/my-flask-app
cloudron install --image devtools.your-domain.com/my-flask-app:latest

# Option B: Embed your code directly in the Dockerfile (no extra files needed)
# Use multi-stage or COPY from a git repo / URL
```

> **Note**: The FastPack Go Deploy CLI is designed for packages generated by FastPackCloudron (which bundle everything into the Dockerfile). For custom apps with additional source files, use the `cloudron` CLI or embed your code in the Docker build context.

## Step 3: Deploy

### Option A: With the Go Deploy CLI (for FastPackCloudron-generated packages)

If your Dockerfile uses `COPY` for files that are in the allow-list (start.sh, nginx.conf, etc.):

```bash
./fastpack-deploy-linux-amd64
# Follow the 4-step wizard
```

### Option B: With the Cloudron CLI (for custom apps with extra source files)

```bash
npm install -g cloudron

# Configure Build Service
cloudron build login \
  --build-service-url https://devtools.your-domain.com \
  --build-token YOUR_BUILD_TOKEN

# Build and push the image
cloudron build --repository devtools.your-domain.com/my-flask-app

# Install the app
cloudron install --image devtools.your-domain.com/my-flask-app:latest
```

## Understanding CloudronManifest.json

Here is the manifest for our Flask app:

```json
{
  "id": "io.fastpack.myflaskapp",
  "title": "My Flask App",
  "version": "1.0.0",
  "manifestVersion": 2,
  "healthCheckPath": "/",
  "httpPort": 8000,
  "description": "A custom Flask application on Cloudron",
  "addons": {
    "localstorage": {},
    "oidc": {}
  }
}
```

| Field | Purpose |
|-------|---------|
| `id` | Unique app identifier (reverse domain notation) |
| `title` | Display name in Cloudron dashboard |
| `httpPort` | The port your app listens on (must match start.sh) |
| `healthCheckPath` | Cloudron pings this path to check if the app is alive |
| `addons.localstorage` | Provides persistent `/app/data/` directory |
| `addons.oidc` | Enables Cloudron SSO — injects `CLOUDRON_OIDC_*` env vars |

## Understanding Cloudron Addons

When you enable addons, Cloudron injects environment variables:

### localstorage

- Provides `/app/data/` — the only writable directory
- Survives app updates and restarts
- Use for database files, uploads, config

### OIDC (SSO)

Environment variables injected:
```
CLOUDRON_OIDC_ISSUER          # OIDC issuer URL
CLOUDRON_OIDC_CLIENT_ID       # Your app's client ID
CLOUDRON_OIDC_CLIENT_SECRET   # Your app's client secret
CLOUDRON_OIDC_DISPLAY_NAME    # Logged-in user's name
```

### PostgreSQL

Environment variables injected:
```
CLOUDRON_POSTGRESQL_HOST
CLOUDRON_POSTGRESQL_PORT
CLOUDRON_POSTGRESQL_USERNAME
CLOUDRON_POSTGRESQL_PASSWORD
CLOUDRON_POSTGRESQL_DATABASE
CLOUDRON_POSTGRESQL_URL       # Full connection string
```

## Common Patterns

### Pattern 1: Static Site (Nginx)

Use image `nginx:latest`, HTTP port 80. Simplest possible custom app.

### Pattern 2: Node.js App

Use image `node:20-alpine`, stack "Node.js". The generated start.sh runs `node server.js`.

### Pattern 3: Background Service (MQTT Broker)

Use image `eclipse-mosquitto:2.0`, set "Web Interface" to No. FastPackCloudron generates a Python healthcheck server.

### Pattern 4: Multi-Service (App + Worker)

Add multiple services in the Advanced section. FastPackCloudron generates an nginx reverse proxy and manages all processes.
